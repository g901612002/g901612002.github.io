Angular2 路由

2017-03-20

1. 呈現路由：

  • 方法：
    • PathLocationStrategy（預設）
      • 採用 HTML5 History API 技術 （只支援到 Internet Explorer 10+ ）
      • 網址結構：http://localhost:4200/dashboard/
    • HashLocationStrategy
      • 採用 HTML4 標準 Hash 網址格式 ( IE6+ )
      • 網址結構：http://localhost:4200/#/dashboard/


2. 路由設定

  • 重要的設定：src/index.html裡面的<basehref="/">
    • 記住！位置一定要放在最前面！(不然資料的位置會隨著url改變而跑掉)
  • 放置元件的地方
    • 路由插座(<router-outlet>)
  • 路由連結套用 Active 樣式：routerLinkActive="css樣式"
  • 避免套用 routerLinkActive 到上層路徑：[routerLinkActiveOptions]="{exact: true}"
  • 導頁：redirectTo
    • 使用 redirectTo 時，一定要加上 patchMatch 屬性
    • 轉向路由最多只能轉向一次，不能從 /aaa 轉到 / 然後再轉向 /bbb
  • 萬用路由：一定要放在最後一個路由定義中!


3.Router 服務元件

當網址發生改變，就會從Router中找出相對應的Route物件，
ex：ActivatedRoute(目前套用的路由)、RouterState(路由狀態)。

  • 可取得完整的路由資訊
    • config
    • events
    • navigated
  • 可透過導覽 API 進行導覽
    • 絕對位址導覽
    • 相對位址導覽 (相對於目前的路由的網址路徑)
  • Router 跟 Component 關係：
    • Component 是不斷的實例化跟摧毀
    • Router 是不斷的存在更新


4. Route 物件

  • path屬性：(必填)
    • 前面不需要加上斜線開頭(/)
    • path加上: 參數名稱來代表一個路由參數
      • 設定路由參數，網址列上就一定要出現該參數值。
      • 如果網址路徑沒有傳入路由參數，該路由會比對失敗。
    • 在 Component 中取得參數(載入：ActivatedRoute, Router)
      • 範例:{ path: ‘hero/:id’, component: HeroDetailComponent }
      • 如果瀏覽器上的網址路徑為 /hero/33 時，路由參數 id 的值就會是 33
  • outlet： 指 component 中有兩個以上的 router-outlet，指定這個router網址是要給哪個 router-outlet 去做顯示。
  • canActivate & canActivateChild： 我能不能進去拜訪這個頁面；假如是 true 就是可以進去，false 就是被拒絕。
  • canDeactivate： 我可不可以離開這個頁面；假如是 true 就是可以離開，false 就是不能。
  • canLoad： 與 loadChildren有關係，建議不要使用。
  • data & resolve： 在進入頁面前，先把資料準備好，才會進入頁面。
    • data屬性：代表預計傳入元件的路由資料
      • 路由參數是從網址路徑傳入ActivatedRoute物件(目前正被啟用的路由物件)
      • 路由資料是從路由定義傳入ActivatedRoute物件
      • 路由資料通常是唯讀/靜態的資料
        • 例如頁面標題、權限角色、麵包屑…
      • 動態的路由：透過resolveguard來取得動態資料


5. Routes 物件

  • 是 Route 物件的集合 (陣列型態)
  • 陣列中的物件是有順序性的 (每個Route(路由)順序很重要)
  • 設定是否允許進入路由 (例如: 實作權限管理) 或是否允許離開路由
